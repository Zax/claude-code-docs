# Linee guida per la documentazione Claude Code

## Progetto
Raccolta di documentazione personale sull'uso di Claude Code, organizzata in file Markdown collegati tra loro.

## Lingua
- Tutto il contenuto deve essere scritto in **italiano**
- Termini tecnici, nomi di comandi, codice e identificatori restano in inglese

## Struttura
- `README.md` in root: indice principale con link a tutti i documenti
- `docs/`: contiene i file di documentazione, uno per argomento
- Questo file (`CLAUDE.md`) NON fa parte della documentazione e non va linkato da essa

## Formato dei file in docs/
- Titolo H1 coerente con l'argomento
- Link di navigazione in testa: `← [Indice](../README.md)` + eventuali precedente/successivo
- Contenuto della sezione
- Commento `<!-- Aggiungi qui le tue scoperte -->` dove opportuno, per segnalare punti di espansione futura

## Convenzioni
- Naming file: kebab-case, estensione `.md` (es. `tips-e-tricks.md`)
- Link tra file: usare percorsi relativi
- Non inventare informazioni: attenersi a fonti ufficiali e verificate
- Mantenere il contenuto conciso e pratico, orientato all'uso quotidiano
